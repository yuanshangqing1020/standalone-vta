# Functional Simulator (C++ / FSIM)

Chinese: [`README_cn.md`](README_cn.md)

This directory is the **C++ functional simulator (FSIM)** for the VTA: it executes `instructions.bin` / `uop.bin` on data in virtual DRAM and produces numerically correct results (no cycle-level timing).

> **Naming:** Older docs refer to a single executable `./vta_simulator`. That binary was split into two entry points. See the Chinese guide [`docs/fsim_nnäøˇfsim_single_layer_cn.md`](../../../docs/fsim_nnäøˇfsim_single_layer_cn.md) for the full picture (including relationship to TSIM and deprecated names).

## What gets built

| Makefile target | Executable | Role |
|-----------------|------------|------|
| `make build/fsim_single_layer` | `build/fsim_single_layer` | Single-layer / multi-layer **independent** VTA ops (`layers_name.csv`) |
| `make build/fsim_nn` | `build/fsim_nn` | **Full network** forward (`dependency.csv`, chaining, CPU ops) |
| `make execute` | (runs `fsim_single_layer`) | Build if needed, then run single-layer FSIM |
| `make nn_execute` | (runs `fsim_nn`) | Build if needed, then run full-network FSIM |

**Do not rely on** `make all` ā†’ `build/functional_simulator`: the target name exists but there is no working link rule. Use the targets above.

Shared backend (linked into both executables): `sim_driver.cc`, `sim_tlpp.cc`, `virtual_memory.cc`, plus `simulator_functions.cc`.

## Prerequisites

- g++ with C++17, `make`, Python 3 (for `config/vta_config.py`)
- Compiler output under repo-root **`compiler_output/`** (relative path from here: `../../../compiler_output/`)

## Building

From this directory:

```bash
make build/fsim_single_layer   # single-layer entry
make build/fsim_nn             # full-network entry
```

From `examples/` (recommended for end-to-end flows):

```bash
make fsim_compile_single_layer   # ā†’ build/fsim_single_layer
make fsim_compile                # ā†’ build/fsim_nn
```

## Running

Must run with **current working directory** = `functional_simulator/` (or use Makefile targets that `cd` here).

**Single layer** (e.g. after `make test_gemm`):

```bash
./build/fsim_single_layer
# or: make execute
```

Reads per-layer files from `compiler_output/`, including `layers_name.csv`, `metadata*.csv`, `input*.bin`, `weight*.bin`, `accumulator*.bin`, `uop*.bin`, `instructions*.bin`. Prints results to stdout by default.

**Full network** (e.g. after `make run`):

```bash
./build/fsim_nn
# or: make nn_execute
```

Also requires `dependency.csv`, `input_nn.bin`; writes `final_output.bin`.

## Typical inputs (`compiler_output/`)

Generated by the VTA compiler (`main_vta_compiler.py`) and/or NN compiler (`vta_backend.py`):

1. **instructions{suffix}.bin** ā€” VTA instruction stream  
2. **uop{suffix}.bin** ā€” micro-op / pointer table  
3. **input{suffix}.bin** ā€” activations (single-layer FSIM; optional suffix e.g. `QLinearConv1`)  
4. **weight{suffix}.bin**, **accumulator{suffix}.bin**, **add_accumulator{suffix}.bin**  
5. **layers_name.csv** ā€” VTA layer list (`nb_vta_ir`, suffix per layer)  
6. **dependency.csv**, **input_nn.bin** ā€” full-network FSIM only  
7. **expected_out.bin** ā€” optional golden for manual compare
